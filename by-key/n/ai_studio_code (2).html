AI Tool Calls — Developer Cheatsheet (This Chat)

A compact, copy-pasteable reference for invoking this chat’s tools via the XML-like <function_calls> syntax. You can batch multiple calls in one block by repeating <invoke>. Aim for ≤ 3 tool calls per turn.

Tip: Tools are for gathering info; plan before you call. Prefer authoritative sources; cite when you’ve used the web.

Favorites

Quick Web Search

<function_calls>
  <invoke>
    <tool_name>web.run</tool_name>
    <parameters>{
      "search_query": [{"q": "latest AI developments", "recency": 7}],
      "response_length": "short"
    }</parameters>
  </invoke>
</function_calls>

Execute Python Code

<function_calls>
  <invoke>
    <tool_name>python.exec</tool_name>
    <parameters>
import math
print(math.sqrt(16))
    </parameters>
  </invoke>
</function_calls>

Generate Image

<function_calls>

    <parameters>{
      "prompt": "A futuristic AI cityscape",
      "size": "1024x1024",
      "n": 1
    }</parameters>
  </invoke>
</function_calls>

Appendix: Perplexity Tool Equivalents

Perplexity tool This chat’s tool Notes
search_web web.run → search_query Use { "search_query": [{"q": "…", "recency": N}] }. Cite with the special cite… tag (not [id]).
fetch_url web.run → open (URL allowed) You can pass a URL as ref_id in open. Use click/find/screenshot as needed.
execute_python python.exec <parameters> is raw code (no JSON wrapper). Private output.
create_chart python_user_visible.exec Use matplotlib only; one chart per figure; don’t set colors. Output is visible to the user.
generate_image image_gen.text2im After generation/edit, reply with an empty message (no summary).
search_files file_search.msearch Include 1–5 focused queries; weave file cites inline using filecite….

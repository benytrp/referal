Perplexity AI — Tool Calls Cheatsheet (Standalone)

A compact, copy-pasteable reference for invoking Perplexity-style tools via the XML-like <function_calls> syntax. Batch multiple calls by repeating <invoke>. Aim for ≤ 3 tool calls per turn.

Tip: Plan first. Use tools to gather facts, then answer. Prefer authoritative sources; cite when you’ve used the web.

Favorites

Quick Web Search

<function_calls>
  <invoke>
    <tool_name>search_web</tool_name>
    <parameters>
      <queries>["latest AI developments"]</queries>
    </parameters>
  </invoke>
</function_calls>

Execute Python Code

<function_calls>
  <invoke>
    <tool_name>execute_python</tool_name>
    <parameters>
      <code>import math; print(math.sqrt(16))</code>
    </parameters>
  </invoke>
</function_calls>

Video

Aligned to your Perplexity Video API (as of 2025-08-09). Field names, enums, and limits match your spec. Include your key as api_key in the body (as required by your endpoints) even if your gateway also injects Authorization.

Base URL (server-side mapping): https://api.perplexity.video/v1/

Tip: Set "async": true on any call to get a job_id immediately, then poll status.

Prompt-to-Video  POST /generate

<function_calls>
  <invoke>
    <tool_name>video.generate</tool_name>
    <parameters>{
      "api_key": "YOUR_API_KEY",
      "prompt": "slow drone fly-through of a neon cyberpunk alley, rain, cinematic",
      "duration": 6,                   
      "fps": 24,                       
      "resolution": "1024x576",      
      "style_preset": "cinematic",   
      "safety_mode": "strict",       
      "async": false
    }</parameters>
  </invoke>
</function_calls>

Edit (inpaint / replace background / extend / loop)  POST /edit

<function_calls>
  <invoke>
    <tool_name>video.edit</tool_name>
    <parameters>{
      "api_key": "YOUR_API_KEY",
      "video_url": "https://example.com/input.mp4",
      "op": "inpaint",                
      "mask": "https://example.com/mask.png",   
      "prompt": "replace billboard with a retro arcade sign, glowing",
      "fps": 24,                        
      "resolution": "1920x1080",
      "async": false
    }</parameters>
  </invoke>
</function_calls>
op enum: "inpaint" | "replace_bg" | "extend" | "loop". For replace_bg, provide bg_url. If duration, fps, or resolution are omitted, the input video values are used.

Compose / Concatenate Clips (with per-clip transition)  POST /compose

<function_calls>
  <invoke>
    <tool_name>video.compose</tool_name>
    <parameters>{
      "api_key": "YOUR_API_KEY",
      "timeline": [
        { "src": "https://ex.com/a.mp4", "in": 0, "out": 4, "transition": "fade", "duration": 0.5 },
        { "src": "https://ex.com/b.mp4", "in": 1, "out": 6 }
      ],
      "fps": 30,
      "resolution": "1080x1080",
      "async": false
    }</parameters>
  </invoke>
</function_calls>
transition enum per clip: "fade" | "cut" | "dissolve". Max total duration 300s.

Function-Calls v2 — Appendix (IDs, Timeouts, Caching, Dependencies)

This appendix adds the v2 envelope and the execution-safety fields you asked for: id, timeout_ms, retry, cache_key, depends_on, invocation_digest (plus optional <expect> constraints) — and the linter rules that govern them.

Annotated Skeleton (two-step chain)

<function_calls version="2" session_id="sess_2025-08-09" turn_id="t42">
  <invoke id="ivk_1" timeout_ms="8000" retry="1" cache_key="2b9f…" invocation_digest="sha256:a3c4…">
    <tool_name>search_web</tool_name>
    <parameters>{"queries":["openai sora licensing news"]}</parameters>
    <expect>freshness<=7d; max_results<=5</expect>
  </invoke>

  <invoke id="ivk_2" timeout_ms="12000" retry="1" cache_key="5f01…" depends_on="ivk_1" invocation_digest="sha256:9a7e…">
    <tool_name>fetch_url</tool_name>
    <parameters>{{ ivk_1.top_urls(3) }}</parameters>
  </invoke>
</function_calls>
